# Security Policy

## Supported versions

There is no supported release yet. Development snapshots are not suitable for
production workloads.

## Reporting a vulnerability

Do not open a public issue for a vulnerability that could expose credentials,
cross volume boundaries, unmount or detach shared storage, or delete data.
Until the public security contact is approved, use the hosting platform's
private vulnerability-reporting feature or contact a repository maintainer
privately. A public release is blocked until a durable public security contact
and response policy are documented.

Never include real credentials, kubeconfigs, project identifiers, filesystem
identifiers, or workload data in a report.
